# Work Log — 2026-06-19

## Feature: Org Letter Template Management

### Summary
Built a production-ready system for organisations to create and manage their own letter templates alongside platform-provided (system) templates. Includes a visual block-based designer, clear Platform vs Org visual differentiation, and hardened backend security.

---

## Backend (`brello_server`)

### Module: `hr-template`

#### Security fixes (critical)

| Issue | Fix |
|-------|-----|
| `GET /:id` had no org-scoping — any authenticated user could read any template or category by UUID | Added `findOrgAccessible(id, orgId)` query to both repositories; controllers now pass `user` to `findOne` |
| `update` / `remove` made two sequential DB reads (TOCTOU race) | Merged into a single `findOrgAccessible` call; ownership checked on that result |
| Category deletion hit FK `RESTRICT` constraint when templates existed | `LetterCategoryService.remove()` now calls `templateRepository.softDeleteByCategory(categoryId, orgId)` before soft-deleting the category |
| `PATCH /:id` with a foreign `category_id` moved templates across orgs | `category_id` on update is now validated with `findOrgAccessible` before applying |

#### New repository methods

**`letter-template.repository.ts`**
- `findOrgAccessible(id, orgId)` — single query: `id = X AND (organization_id = orgId OR is_system = true) AND is_deleted = false`
- `softDeleteByCategory(categoryId, orgId)` — bulk soft-delete of all org-owned (non-system) templates in a category
- `update(id, data, orgId)` — now accepts `orgId` and re-fetches via `findOrgAccessible`

**`letter-category.repository.ts`**
- `findOrgAccessible(id, orgId)` — same pattern as above

#### Service hardening

- `assertOrgContext(user)` guard on every method — returns `400` if `user.organizationId` is missing (e.g. platform admin token)
- `extractVariables` regex tightened to `\w+` — only captures word-character variable names, prevents whitespace keys
- `LetterCategoryService` now injects `LetterTemplateRepository` for cascade deletion

#### Document types rationalised

Removed `payslip` (auto-generated by payroll, not a letter template), `certificate`, and `appraisal` tabs — both overlap with categories already seeded under HR Letters.

**Final document types:** `hr_letter` · `onboarding` · `notice`

Removed types also cleaned from `DOCUMENT_TYPES` const and `CreateLetterCategoryDto` `@IsIn` guard.
